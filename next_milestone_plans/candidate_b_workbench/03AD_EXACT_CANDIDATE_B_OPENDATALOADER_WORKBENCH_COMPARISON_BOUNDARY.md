# 03AD - Exact Candidate B OpenDataLoader Workbench Comparison Boundary

## Purpose

State exactly what Candidate B is allowed to do.

## A. Exact boundary statement

Candidate B is a tests/report-only side-by-side comparator that:
- consumes the existing manifest-driven lower-layer NRC APS corpus
- runs OpenDataLoader locally through the Python wrapper only
- derives structural and layout comparison summaries
- compares those summaries against current merged-baseline lower-layer outputs and invariants
- may optionally compare against the exact frozen Candidate A workbench artifact and related admitted Candidate A anchors
- emits workbench reports under `tests/reports/...`
- changes nothing about runtime connector behavior

## B. Exact inputs

Candidate B consumes:
- the existing corpus manifest and current fixture PDFs
- the current lower-layer outputs generated by `nrc_aps_document_processing.process_document(...)`
- the current lower-layer invariants encoded in `tests/test_nrc_aps_document_processing.py`
- optional exact Candidate A anchors frozen in `04B`

## C. Exact outputs

Candidate B may emit only:
- raw OpenDataLoader JSON, Markdown, or image outputs under a run-scoped tests-side raw-output root
- a Candidate B proof report
- a Candidate B compare report
- a Candidate B retention manifest
- optional labels or manifest sidecars under the existing corpus fixture folder

## D. Explicit non-goals

Candidate B does not:
- become a connector mode
- become a new selector
- become a new runtime parser path
- replace the current lower layer
- change OCR or media routing
- change `visual_page_refs` semantics
- alter artifact ingestion
- alter review/runtime surfaces
- alter endpoints, persistence, or run-detail refs
- reopen the closed PageEvidence lane

## E. Limitation controls that must remain explicit

Candidate B must explicitly carry these limitation classes:
- vector-heavy pages already governed by the current owner path
- scanned OCR-dependent pages
- visual-preservation pages already governed by the current owner path and admitted Candidate A path
- cases where OpenDataLoader semantic richness is not program-relevant

If Candidate B claims wins on those classes without a separate proof and scope change, that is a planning violation.
