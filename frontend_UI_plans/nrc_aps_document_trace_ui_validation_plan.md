## 6. Test Matrix

### 6.1 Service Tests

Verify:

- trace assembly resolves `connector_run_target`, `aps_content_linkage`, `aps_content_document`, and `aps_content_chunk` correctly
- trace assembly exposes explicit missingness when one or more optional layers are absent
- diagnostics `ordered_units` are used for extracted-unit payloads
- `aps_content_units_v2` is not misclassified as the authoritative fine-grained unit source
- deduplicated `content_id` states do not collapse `target_id`

### 6.2 API Tests

Verify:

- all document-trace routes are GET-only
- the source route rejects path traversal and non-bounded access
- the trace manifest payload matches the frozen contract
- missing normalized text or diagnostics return stable unavailability states
- retrieval absence does not break trace manifest generation

### 6.3 Page Tests

Verify:

- the document-trace page shell is served by the backend
- run selector and document selector placeholders are present
- tab labels are present
- the existing review page remains available

### 6.4 Review Regression Tests

Verify:

- the current three review modes still load
- the current three review mode labels remain unchanged
- theme control still works
- graph zoom preservation still works
- current tree and details-drawer behavior still works
- current file-preview behavior still works
- existing review page routes and payloads are unchanged

## 7. Representative Validation Scenarios

Manual and automated validation together must cover at least these significant scenarios:

- a strong PDF trace with source blob, diagnostics, normalized text, and indexed chunks all present
- a deduplicated-content scenario where multiple targets share one `content_id`
- a missing-layer scenario where one or more optional layers are unavailable
- a retrieval-absent runtime where trace functionality still succeeds
- a non-PDF or unsupported-source fallback scenario

## 8. Required Edge-Case Tests

The implementation must include tests for:

- target exists but linkage row is missing
- linkage row exists but source blob file is missing on disk
- diagnostics are missing
- normalized text is missing
- indexed chunks are missing
- downstream usage is unavailable because the target was not selected downstream
- non-PDF source type falls back to text-like or unsupported rendering state
- multiple targets share one `content_id`

## 9. Read-Only Invariant Tests

The implementation must prove:

- no route uses POST, PUT, PATCH, or DELETE
- no route triggers regeneration, seeding, or repair code
- empty or partial trace states do not generate artifacts
- source streaming remains bounded and non-mutating

## 10. Manual Verification Checklist

After automated tests pass, manual verification should confirm:

- the document-trace page opens with the selected run
- the document selector shows run-scoped targets
- a PDF source renders in the left pane from the run blob
- `Summary` tab shows identity and completeness correctly
- `Extracted Units` tab shows page-scoped units from diagnostics
- clicking an extracted unit moves the source viewer to the right page
- changing the source page highlights or filters the right pane to the same page
- `Normalized Text` works when present and degrades clearly when absent
- `Indexed Chunks` works when present
- `Diagnostics` surfaces quality/degradation information
- `Downstream Usage` remains empty or conditional when no attributable usage exists
- light and dark themes both remain legible
- the current `/review/nrc-aps` page still behaves as before
- the current review page still preserves zoom state on node/file interaction
- the current review page still supports the existing drawer and tree interaction flow

## 12. Pass Criteria

The feature passes only if:

- a real PDF source renders from run storage
- the trace manifest works without retrieval rows
- page-level synchronization works on a real document
- missing layers are explicit rather than silent
- the existing review UI is not regressed
- the implemented page remains materially aligned with the requested two-pane source-versus-processed inspection concept from the user’s mockup and description
- all focused and regression tests pass

## 13. Fail Criteria

Stop and re-scope if:

- the implementation depends on the current local corpus path
- exact region-level sync is being promised without supporting anchors
- retrieval becomes a required dependency
- the right pane collapses extraction, diagnostics, and downstream usage into one untyped payload
- the current review page is being overloaded instead of using the dedicated page
