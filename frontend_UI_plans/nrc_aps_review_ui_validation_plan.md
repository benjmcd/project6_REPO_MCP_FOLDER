## 3. Validation Goals

The implementation must prove:

- review run discovery works
- reviewable and non-reviewable runs are classified correctly
- the canonical graph and run-specific graph build consistently
- the strict filesystem tree stays within bounded roots
- node and file details are stable and useful
- the UI remains read-only
- known mismatches degrade visibly rather than crashing

## 5. Test Matrix

### 5.1 Catalog Tests

Verify:

- latest completed reviewable NRC APS run is selected as the default
- non-reviewable NRC APS runs can be returned as disabled rows with reasons
- unrelated connector families are excluded from the NRC APS selector
- review roots are discovered only from allowlisted locations

### 5.2 Graph Tests

Verify:

- both views consume the same canonical graph registry
- all frozen node ids exist
- all frozen edges exist
- the run-specific graph for the golden runtime materializes expected node states
- the general view stays less dense than the run-specific view

### 5.3 Tree Tests

Verify:

- the tree is a strict filesystem tree
- the tree stays bounded to allowlisted review roots
- tree ids are deterministic and stable
- mapped files for a selected node can be auto-revealed
- no arbitrary path traversal is possible

### 5.4 Details Tests

Verify:

- node details return metadata, stage context, warnings, and small structured summaries
- file details return metadata, file class, mapped node id, and structured summaries
- no file details endpoint returns a raw file body in v1

### 5.5 API Tests

Verify:

- all review endpoints are GET-only
- payloads conform to the frozen response models
- missing reviewable runs return the normal shell state inputs needed for disabled panes
- disabled non-reviewable runs carry reason codes

### 5.6 Page Tests

Verify:

- the UI route is served by the backend
- the page loads the shell even when no reviewable run exists
- the page can default to the latest reviewable run when one exists
- the details drawer placeholder area is present

## 7. Required Edge-Case Tests

The implementation must include tests for:

- no reviewable runs
- disabled run with `missing_review_summary_root`
- disabled run with `run_not_completed`
- reviewable run with optional downstream nodes marked `not_exercised`
- selected run with missing advertised report refs on disk
- known golden-run mismatch on:
  - `artifact_dedup_report`
  - `checkpoint_history`
  - `event_log`

## 8. Read-Only Invariant Tests

The review implementation must prove:

- no route uses POST, PUT, PATCH, or DELETE in v1
- no route calls execution, regeneration, or seeding code
- empty review discovery does not seed or generate artifacts
- file browsing remains bounded and non-mutating

## 9. Manual Verification Checklist

After automated tests pass, manual verification should confirm:

- the selector opens the latest reviewable run by default
- disabled runs are visually disabled with reasons when present
- the `Pipeline Overview` is cleaner than the `Run-specific Overview`
- clicking a diagram node highlights mapped files
- clicking a tree node highlights the mapped diagram node when available
- the right-side details drawer shows metadata plus structured summary, not raw preview
- the shell still loads with disabled panes when no reviewable run exists

## 11. Failure Standard

The implementation should be considered incomplete if any of the following occur:

- graph nodes are inferred ad hoc instead of coming from the canonical registry
- tree traversal escapes the allowlisted review roots
- the page depends on a write path or artifact generation side effect
- known mismatches crash the overview or hide required nodes
- selector semantics differ from the frozen v1 defaults
