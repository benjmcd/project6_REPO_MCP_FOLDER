## 2. Handoff Bridge (`index_builder.py`)
The bridge processes Phase 7A result JSONs and populates the local database with the accepted Phase 7A outputs, creating APS content-table rows intended for downstream consumers (including Evidence Bundle).

### 2.1 Bootstrapping Strategy
To maintain referential integrity in a developer-only/standalone environment, the bridge implements a **Synthetic Run Strategy**:
- It creates a placeholder `ConnectorRun` (ID: provided by caller).
- It creates a placeholder `ConnectorRunTarget` per file.
- This ensures downstream analytical services can query the DB without requiring the full ScienceBase/MCS connector stack to be active.

## 3. Mapping Schema

### 3.1 Document Mapping (`aps_content_document`)
| Source Field (Phase 7A JSON) | Target Column (DB) | Notes |
| :--- | :--- | :--- |
| `accession_number` | `content_id` | Primary identity. Fallback to `metadata` or Path. |
| N/A | `content_contract_id` | Hardcoded: `aps_content_units_v2`. |
| N/A | `chunking_contract_id` | Hardcoded: `aps_chunking_v2`. |
| `normalization_contract_id` | `normalization_contract_id` | e.g. `aps_text_normalization_v2`. |
| `normalized_text` | `normalized_text_sha256` | SHA256 of the aggregated text. |
| `normalized_text` length | `normalized_char_count` | Total character count. |
| `ordered_units` length | `chunk_count` | Number of logical segments. |
| `document_class` | `document_class` | e.g., layout_complex_pdf. |
| `page_count` | `page_count` | Extracted from PDF metadata. |

### 3.2 Chunk Mapping (`aps_content_chunk`)
| Source Field (Phase 7A JSON) | Target Column (DB) | Notes |
| :--- | :--- | :--- |
| Item in `ordered_units` | `chunk_ordinal` | 0-indexed position (canonical ordering). |
| `chunk_id` (derived) | `chunk_id` | Generated by canonical chunk ID helper; stable hash based on content ID, ordinal and chunk text checksum. |
| `unit.text` | `chunk_text` | Actual content slice. |
| `unit.text` hash | `chunk_text_sha256` | For integrity checks. |
| `unit.start_char` | `start_char` | Global index in full text. |
| `unit.end_char` | `end_char` | Global index in full text. |

### 3.3 Linkage Mapping (`aps_content_linkage`)
| Source Field | Target Column | Notes |
| :--- | :--- | :--- |
| Run ID (provided) | `run_id` | e.g., `run_20260314_010136`. |
| Target ID (resolved) | `target_id` | FK to `connector_run_target` (Synthetic). |
| Artifact Path | `content_units_ref` | Path to the Phase 7A `result.json`. |
| Text Asset Path | `normalized_text_ref` | Path to extracted `text.md` file. |

## 4. Integrity Constraints
- **Source Provenance:** Every `ApsContentLinkage` must point to a valid file in the Phase 7A report directory.
- **Deterministic IDs:** Re-running the bridge on the same files must produce identical IDs and hashes.
- **Foreign Key Safety:** As defined in 2.1, the bridge ensures referential integrity via synthetic records.
